# 总体设计

## 1. 架构目标

BeamDrop 的第一阶段采用 Client/Server 架构：

```text
Client  -- TCP --  Server
```

设计重点：

- 模块职责单一；
- 协议层与网络层解耦；
- 传输流程与 CLI 解耦；
- 便于后续加入 UDP 发现、FastAPI 管理 API、Web 控制台。

## 2. 模块划分

```text
src/app          应用编排层
src/cli          命令行解析
src/network      TCP 连接、监听、读写
src/protocol     Packet、序列化、解析、校验
src/transfer     文件发送、接收、续传、进度
src/filesystem   文件扫描、路径处理、读写封装
src/config       配置加载与默认值
src/logger       日志封装
src/utils        通用工具
```

## 3. 关键类规划

### network

```text
Socket
TcpServer
TcpClient
```

职责：只处理连接、监听、字节读写，不理解文件和业务协议。

### protocol

```text
PacketHeader
Packet
PacketType
Serializer
PacketParser
```

职责：将结构化消息转换为网络字节流，并进行 magic、version、length、checksum 校验。

### transfer

```text
Sender
Receiver
TransferSession
ResumeManager
ProgressReporter
```

职责：组织文件传输流程，处理断点续传和进度上报。

### filesystem

```text
FileScanner
FileReader
FileWriter
PathUtils
```

职责：扫描文件、生成相对路径、创建目录、防止路径穿越。

### app

```text
ServerApp
ClientApp
```

职责：组合配置、日志、网络、协议、传输模块，提供应用入口。

## 4. 数据流

发送流程：

```text
CLI -> ClientApp -> FileScanner -> Sender -> Protocol -> TcpClient -> Network
```

接收流程：

```text
Network -> TcpServer -> Protocol -> Receiver -> FileWriter -> 保存目录
```

## 5. 后续扩展点

- UDP 自动发现：新增 `discovery/` 模块；
- HTTP 管理 API：复用 `transfer` 状态，暴露给 FastAPI 或 C++ HTTP 服务；
- Web 控制台：通过 API 展示设备、任务、日志；
- 加密认证：扩展 HELLO 阶段与 Packet flags。
