# 需求分析

## 1. 项目背景

BeamDrop（邻光）是一个局域网文件传输工具，目标是在同一局域网内快速、可靠地从一台设备向另一台设备发送文件或文件夹。

本项目短期目标不是复刻 AirDrop，而是完成一个结构清晰、可演示、可扩展的 C++ 命令行文件传输工具。

## 2. 目标用户

- 同一局域网内需要快速传文件的个人用户
- 需要课程项目演示的开发者
- 希望使用 CLI 自动化传输文件的用户

## 3. MVP 范围

### 必须支持

| 功能        | 说明                                   |
| ----------- | -------------------------------------- |
| 单文件发送  | Client 通过 TCP 向 Server 发送一个文件 |
| 多文件发送  | 一次命令发送多个路径                   |
| 文件夹发送  | 递归扫描目录，保留相对路径结构         |
| 进度显示    | 显示当前文件进度、总进度、速度         |
| SHA256 校验 | 发送前计算 hash，接收后校验一致性      |
| 断点续传    | 传输中断后从已完成 offset 继续         |
| 配置文件    | 管理端口、保存目录、日志级别、分块大小 |
| 日志        | 输出运行状态、错误、传输结果           |

### 暂不支持

| 功能         | 原因                                 |
| ------------ | ------------------------------------ |
| GUI          | 会明显拉长开发周期                   |
| Web 控制台   | 等 CLI 核心稳定后再做                |
| UDP 自动发现 | 作为加分项，不阻塞 MVP               |
| 加密认证     | 第一阶段可预留设计，不作为硬性完成项 |
| 跨平台安装包 | 后续工程化阶段处理                   |

## 4. 成功标准

MVP 完成时应满足：

1. 能启动接收端。
2. 能发送单文件、多文件、文件夹。
3. 接收端能按相对路径保存文件。
4. 传输完成后 SHA256 校验通过。
5. 中断后可续传。
6. 有清晰 CLI 帮助信息。
7. 有日志和配置文件。
8. 有基本单元测试和至少一个端到端测试。
