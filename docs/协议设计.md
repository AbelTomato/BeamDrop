# 协议设计

## 1. 设计目标

BeamDrop 协议用于在 TCP 字节流上区分不同类型的消息，并保证 payload 长度和校验可控。

第一阶段采用：

```text
固定二进制 Header + JSON 或二进制 Payload
```

## 2. Packet Header

```text
Magic       4 bytes   固定为 "BDRP"
Version     2 bytes   协议版本，MVP 为 1
Type        2 bytes   消息类型
Flags       4 bytes   标志位
Length      8 bytes   payload 长度
Checksum    32 bytes  payload SHA256；不需要时填 0
```

Header 总长度：52 bytes。

## 3. Packet Type

| 类型       | 说明                |
| ---------- | ------------------- |
| HELLO      | 握手与传输 manifest |
| FILE_INFO  | 文件或目录元信息    |
| RESUME_REQ | 请求续传位置        |
| RESUME_ACK | 返回续传 offset     |
| DATA       | 文件数据块          |
| FILE_END   | 单个文件结束        |
| FINISH     | 整个传输任务结束    |
| ERROR      | 错误消息            |
| HEARTBEAT  | 保活，后续可选      |

## 4. HELLO Payload / Transfer Manifest

当前实现使用文本 manifest 作为会话头 payload：

```text
beamdrop-manifest-v1
file_count=3
total_bytes=123456
```

字段：

| 字段          | 说明                          |
| ------------- | ----------------------------- |
| version line  | 固定为 `beamdrop-manifest-v1` |
| `file_count`  | 本次会话要发送的文件数量      |
| `total_bytes` | 本次会话全部文件总字节数      |

接收端必须拒绝旧的纯文本文件数量 payload、未知版本和非法数字字段。

## 5. FILE_INFO Payload

当前实现使用 JSON：

```json
{
  "relative_path": "docs/a.txt",
  "size": 123456,
  "sha256": "..."
}
```

字段 `relative_path` 和 `sha256` 必须为非空字符串；`size` 必须为无符号整数。接收端必须拒绝旧的临时换行文本格式。

## 6. 典型交互流程

### 单文件发送

```text
Client -> Server: HELLO(manifest: file_count, total_bytes)
Client -> Server: FILE_INFO
Server -> Client: RESUME_ACK(offset=0)
Client -> Server: DATA...
Client -> Server: FILE_END
Client -> Server: FINISH
```

### 断点续传

```text
Client -> Server: FILE_INFO(size, sha256, relative_path)
Server 检查本地 transfer_state.json 和已存在文件
Server -> Client: RESUME_ACK(offset=n)
Client 从 offset=n 开始发送 DATA
Client -> Server: FILE_END
Server 校验最终文件 size 和 SHA256
```

`RESUME_ACK` payload 使用 JSON：

```json
{"offset": 123}
```

`offset` 必须是无符号整数，且满足 `0 <= offset <= FILE_INFO.size`。接收端计算 offset 时必须绑定 `relative_path + size + sha256`，并在状态 offset 与本地文件大小冲突时选择保守 offset，不能跳过未确认写入的数据。

### 多文件/目录发送

```text
Client -> Server: HELLO(manifest: file_count=N, total_bytes=M)
Client -> Server: FILE_INFO #1
Server -> Client: RESUME_ACK #1
Client -> Server: DATA... #1
Client -> Server: FILE_END #1
...
Client -> Server: FILE_INFO #N
Server -> Client: RESUME_ACK #N
Client -> Server: DATA... #N
Client -> Server: FILE_END #N
Client -> Server: FINISH
```

## 7. 校验策略

- `DATA` 包可选择计算 payload checksum；
- 文件整体必须用 SHA256 校验；
- 接收端在 `FILE_END` 后计算文件 SHA256；
- 校验失败时发送 `ERROR` 并记录日志。

## 8. 错误处理

ERROR payload：

```json
{
  "code": "CHECKSUM_MISMATCH",
  "message": "received file sha256 mismatch"
}
```
