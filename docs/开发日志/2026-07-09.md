## 2026-07-11 11:38 - 应用层：抽出 app service 并迁移 CLI

### 实现内容

- 新增 `beamdrop::app::SendService`、`beamdrop::app::ReceiveService` 和 `beamdrop::app::TransferTask`。
- `SendService` 封装发送会话编排：文件扫描、连接、manifest 构造、`Sender` 调用、结束包发送、请求校验和进度回调转换。
- `ReceiveService` 封装接收会话编排：接收 `HELLO`/`FINISH` 包、manifest 解码、`Receiver` 调用、请求校验和进度回调转换。
- `beamdrop send` / `beamdrop serve` 已迁移为调用 app service，`src/main.cpp` 中重复的协议编排、`Sender` / `Receiver` 直接调用和底层进度转换已移除。
- CLI 进度输出改为消费 `beamdrop::app::TransferProgress`，保持原有命令行交互行为。

### 测试内容

- 新增 `app_send_service_tests`，覆盖发送请求参数校验与取消令牌状态。
- 新增 `app_receive_service_tests`，覆盖接收请求参数校验与取消令牌状态。
- 新增 `app_service_single_file_transfer_tests`，覆盖 app service 层单文件端到端发送/接收。
- 执行本机 CLI loopback smoke test：`beamdrop serve` 与 `beamdrop send` 通过 `127.0.0.1` 完成单文件传输，进度输出和接收文件内容均正常。

### 结果

通过。CLI 已复用 app service，核心协议编排不再散落在 `main.cpp` 中。
