## 2026-07-02 22:45 - 断点续传 Slice 3：Receiver 续传协商与追加写入

### 实现内容

- 扩展 `transfer::Receiver` 构造函数，新增续传配置：
  - `enable_resume`
  - `state_file`
- `Receiver` 收到 `FILE_INFO` 后统一回复 `RESUME_ACK(offset)`。
- 未启用续传时回复 `offset=0`，保持覆盖写入路径。
- 启用续传时通过 `ResumeManager` 查询状态 offset，并结合本地已存在文件大小和 `FILE_INFO.size` 计算可信 offset。
- offset 冲突时采用保守策略：取状态 offset、本地文件大小、文件总大小三者最小值，避免跳过未确认写入的数据。
- offset 为 0 时以 `std::ios::trunc` 覆盖写入。
- offset 大于 0 时先将本地文件截断到可信 offset，再以 `std::ios::app` 追加写入。
- 每收到一个 `DATA` packet 后更新状态 offset。
- 收到 `FILE_END` 后校验最终文件大小和完整文件 SHA256。
- 校验通过后清理对应续传状态记录。
- 更新 CMake，使单文件和目录传输集成测试目标链接 `ResumeAckCodec` 与 `ResumeManager`。

### 验证内容

- 调整 `single_file_transfer_tests`，启用 Receiver resume 配置并验证 `RESUME_ACK(offset=0)`。
- 调整 `directory_transfer_tests`，启用 Receiver resume 配置并验证每个文件均返回 `RESUME_ACK(offset=0)`。
- 验证传输完成后状态记录为空。
