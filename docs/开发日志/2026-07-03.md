## 2026-07-03 10:22 - 大文件传输性能问题分析与优化计划

### 实现内容

- 新增 `docs/大文件传输性能优化计划.md`，记录 `5.9GB` 文件传输约 `8 分半`、`iperf3` 约 `340MB/s`、传输开始前长停顿、断点续传间隔偏久等现象。
- 明确当前更直接的性能瓶颈不是写入缓冲区时间过长，而是：
  - `src/transfer/Sender.cpp` 在发送 `FILE_INFO` 前完整计算 SHA-256，导致大文件传输前同步阻塞；
  - `src/transfer/Receiver.cpp` 接收完成后再次整文件 SHA-256 校验，增加总耗时；
  - `src/transfer/Receiver.cpp` 每个 Data 包都调用 `ResumeManager::update_offset()`；
  - `src/transfer/ResumeManager.cpp` 每次更新状态都会读取并重写 JSON 状态文件，默认 `1MiB` chunk 下 `5.9GB` 文件约产生数千次状态文件读写；
  - Data 包发送路径存在额外内存复制；
  - 进度输出频率可能进一步影响 Windows 控制台环境下的吞吐，后续需要实现阶段确认。
- 制定分阶段优化计划：
  - P0：断点续传状态写入节流、进度输出节流、大文件 chunk size 建议；
  - P1：流式哈希或配置化校验，消除发送前预哈希停顿；
  - P2：优化 Data 包写入路径，减少大 payload 内存复制；
  - P3：补充性能基准和回归验证流程。
