# 2026-07-14 开发日志

## Slice 2：pybind11 薄绑定补全与 Windows 验证

### 实现内容

- 为 `beamdrop_native.send(...)` 增加可选 `on_progress` Python 回调。
- 发送任务执行期间使用 `py::gil_scoped_release` 释放 GIL；C++ 进度回调调用 Python 函数前使用 `py::gil_scoped_acquire` 恢复 GIL。
- Python 回调抛出 `py::error_already_set` 时，将其转换为 binding 内部的 `PythonProgressCallbackError`；C++ `SendService` 将其归一为 `ServiceResult` 的 `ErrorCode::InternalError`，binding 最终抛出 `BeamDropError`。
- 新增 `ReceiverService` Python 持久对象，适配 `ReceiveServerService::start/status/stop`。Python 持有对象期间，接收服务持续监听；不暴露裸 socket、`TcpServer` 或 `Receiver`。
- 新增 `ReceiverState`、`ReceiverStatus`，并将接收服务的直接失败继续转换为带稳定错误码的 `BeamDropError`。
- 新增 `bindings/python/tests/test_beamdrop_native.py`，覆盖真实 loopback、进度回调、中文文件名、SHA256、回调异常和非法回调参数。
- 在 `bindings/python/CMakeLists.txt` 中新增 `beamdrop_python_binding_tests` CTest 条目，并通过 target 输出目录设置 `PYTHONPATH`。

- 同进程 loopback 成功传输 4 MiB 文件；发送进度收到：`TASK_STARTED`、多次 `TRANSFERRING`、`FILE_COMPLETED`、`TASK_COMPLETED`。
- loopback 接收文件 SHA256 与源文件一致。
- Python 回调故意抛出 `RuntimeError` 后，调用方收到 `BeamDropError(code=ErrorCode.INTERNAL_ERROR)`，Python 进程未崩溃。
- pytest 已通过，且真实使用中文文件名 `数据.bin` 验证 UTF-8 路径边界。
- 重新配置并执行 Windows CTest 后，`beamdrop_unit_tests`、`beamdrop_integration_tests`、`beamdrop_python_binding_tests` 共 3/3 通过；说明 CTest 能通过 `PYTHONPATH=$<TARGET_FILE_DIR:beamdrop_native>` 调用 pytest 并加载当前构建的 native 模块。

## Slice 4：TaskManager / ReceiverManager 应用层基础

### 实现内容

- 新增后端内存领域 DTO：`StartReceiverRequest`、`ReceiverSnapshot`，继续将 HTTP/Pydantic wire model 延后至 Slice 5。
- 实现线程安全 `TaskManager`：任务以宿主生成的 UUID 登记，状态只允许 `pending -> running -> completed/failed`；发送在 `asyncio.to_thread()` 中执行，不阻塞 event loop。
- 进度回调通过同一把 `threading.Lock` 更新运行中任务快照；任务已进入 `completed` 或 `failed` 后，迟到进度被忽略，不会覆盖终态。
- `NativeGatewayError` 映射为原结构化错误；未知 gateway 异常映射为 `INTERNAL_ERROR` 的失败快照，避免任务永久停在 `running`。
- 新增 `ReceiverManager`：维护单例接收服务的本地快照，支持启动、状态查询、幂等停止，以及重复启动时的确定性错误；native 启动或停止异常转换为 `FAILED` 快照。
- 扩展 `NativeGateway` / `PybindNativeGateway`，适配 pybind `ReceiverService.start/status/stop`；`PybindNativeGateway` 持有一个持久 `ReceiverService` 实例，不把 pybind DTO 暴露给上层。
- 扩展 `FakeNativeGateway`，为发送任务和接收服务状态机提供可控且不依赖真实 socket 的单元测试替身。
- `ReceiverManager` 额外使用操作锁串行化 `start/stop`，避免阻塞 native 调用返回时被并发操作覆盖快照。
