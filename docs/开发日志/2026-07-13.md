## 2026-07-13 22:19 - Slice 2 薄绑定：领域 DTO、结构化异常与同步发送

### 实现内容

- 将 C++ 领域枚举暴露为 Python API：`ErrorCode`、`TransferDirection`、`TransferStage`。
- 将 `ServiceError`、`SendResult`、`TransferProgress` 暴露为只读 Python 值对象；Python 侧只能读取 C++ 产生的快照，不能构造或修改这些结果对象。
- 新增模块级 `BeamDropError`，继承 Python `RuntimeError`；C++ `ServiceResult` 失败时由 binding 转换为该异常，并通过 `error.code` 保留机器可识别的 `ErrorCode`，不要求上层解析错误文本。
- 新增同步 `beamdrop_native.send(paths, host, port, chunk_size=...)`：将 Python `list[str]` 转换为 C++ `SendRequest`，调用 `SendService::send(...)`，成功时返回只读 `SendResult`。
- 在绑定边界校验端口的 `uint16_t` 范围；领域级空路径、空 host、端口为 0、chunk size 为 0 等错误仍由 C++ `SendService` 统一验证并返回 `BeamDropError(code=INVALID_REQUEST)`。

### 验证内容

```powershell
cmake --build build/windows-msvc2026-python --config Debug
```

```powershell
$pyd = Get-ChildItem build\windows-msvc2026-python -Recurse -Filter 'beamdrop_native*.pyd' | Select-Object -First 1
$env:PYTHONPATH = $pyd.DirectoryName
& "$PWD\.venv\Scripts\python.exe" -c "import beamdrop_native as b; try: b.send([], '127.0.0.1', 9000); except b.BeamDropError as e: print(e.code, str(e))"
Remove-Item Env:PYTHONPATH
```

### 测试结果

- 模块重新构建与导入成功。
- Python 可读取 `ErrorCode.NETWORK_FAILURE`、`TransferDirection.SEND`、`TransferStage.TRANSFERRING`。
- 空路径发送请求得到 `BeamDropError`，并可读取稳定错误码 `ErrorCode.INVALID_REQUEST`。
- 已按用户反馈完成同步发送入口的验证；尚未将其作为真实 loopback 传输或回调验证的证据。
