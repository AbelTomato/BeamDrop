# 命令行设计

## 1. 命令名

统一使用：

```bash
beamdrop
```

## 2. serve

当前状态：**已实现最小可用版本**。

启动接收端：

```bash
beamdrop serve
```

带参数：

```bash
beamdrop serve --config ./config/beamdrop.example.json --host 0.0.0.0 --port 9090 --save-dir ./received --log-file ./logs/beamdrop.log
```

参数：

| 参数         | 说明         | 默认值              |
| ------------ | ------------ | ------------------- |
| `--host`     | 监听地址     | `0.0.0.0`           |
| `--port`     | 监听端口     | `9090`              |
| `--save-dir` | 保存目录     | `received`          |
| `--log-file` | 日志文件路径 | `logs/beamdrop.log` |
| `--config`   | 配置文件路径 | 无                  |

当前限制：

- 只接收一个 TCP 连接；
- 只处理一次传输会话；
- 会话完成后进程退出。

## 3. send

当前状态：**已实现最小可用版本**。

发送单文件：

```bash
beamdrop send ./movie.mp4 --to 192.168.1.23:9090
```

发送多文件/文件夹：

```bash
beamdrop send ./a.txt ./docs --to 192.168.1.23:9090
```

参数：

| 参数             | 当前状态 | 说明                         |
| ---------------- | -------- | ---------------------------- |
| positional paths | 已实现   | 待发送文件或目录，可传多个   |
| `--to`           | 已实现   | 目标地址，格式为 `host:port` |
| `--log-file`     | 已实现   | 日志文件路径                 |
| `--config`       | 已实现   | 读取配置文件                 |
| `--chunk-size`   | 已实现   | 指定 DATA 分块大小           |

当前限制：

- 尚未支持 `--resume`；
- 目录会被递归扫描为文件列表；空目录不会被单独创建。

配置覆盖顺序：默认值 < `--config` 文件 < 命令行显式参数。

当前进度输出：

```text
beamdrop send [1/2] a.txt 4/5 bytes
beamdrop send [1/2] a.txt 5/5 bytes done
beamdrop receive [1/2] a.txt 5/5 bytes done
```

说明：

- `send` 在每个 DATA chunk 发送后输出当前文件进度；
- `serve` 在每个 DATA chunk 写入后输出当前文件进度；
- 当前输出为基础逐行文本，尚未做速度、剩余时间或单行刷新。

当前文件日志：

```text
2026-06-30 13:31:57 [INFO] send starting endpoint=127.0.0.1:19095 paths=build-mingw/log-e2e/send
2026-06-30 13:31:57 [INFO] send completed file_count=2
2026-06-30 13:31:57 [INFO] serve starting host=127.0.0.1 port=19095 save_dir=build-mingw/log-e2e/received
2026-06-30 13:31:57 [INFO] serve completed file_count=2
```

说明：

- `send` 记录扫描、连接、完成和错误；
- `serve` 记录启动、接收连接、文件数量、完成和错误；
- 当前日志为追加写入的文本文件，尚未支持日志级别过滤。

当前内部会话边界：

- 发送端先发送 `HELLO` packet；
- `HELLO` payload 使用 `beamdrop-manifest-v1` 文本 manifest；
- manifest 当前包含 `file_count` 和 `total_bytes`；
- 随后连续发送每个文件的 `FILE_INFO` / `DATA` / `FILE_END`；
- 最后发送 `FINISH` packet。

manifest 示例：

```text
beamdrop-manifest-v1
file_count=2
total_bytes=14
```

## 4. config

当前状态：**已实现基础配置功能**。

生成默认配置：

```bash
beamdrop config init --path ./config/beamdrop.json
```

查看配置：

```bash
beamdrop config show --path ./config/beamdrop.json
```

参数：

| 参数                 | 说明         | 默认值                 |
| -------------------- | ------------ | ---------------------- |
| `--path`, `--config` | 配置文件路径 | `config/beamdrop.json` |

当前行为：

- `config init` 写入默认配置，并自动创建父目录；
- `config show` 读取配置文件并输出规范化 JSON；
- `--path` 和 `--config` 在 `config` 子命令中等价。
