# 命令行设计

## 1. 命令名

统一使用：

```bash
beamdrop
```

## 2. serve

当前状态：**已实现最小可用版本**。

启动接收端：

```bash
beamdrop serve
```

带参数：

```bash
beamdrop serve --host 0.0.0.0 --port 9090 --save-dir ./received
```

参数：

| 参数         | 说明     | 默认值     |
| ------------ | -------- | ---------- |
| `--host`     | 监听地址 | `0.0.0.0`  |
| `--port`     | 监听端口 | `9090`     |
| `--save-dir` | 保存目录 | `received` |

当前限制：

- 只接收一个 TCP 连接；
- 只处理一次传输会话；
- 会话完成后进程退出；
- 尚未支持 `--config`。

## 3. send

当前状态：**已实现最小可用版本**。

发送单文件：

```bash
beamdrop send ./movie.mp4 --to 192.168.1.23:9090
```

发送多文件/文件夹：

```bash
beamdrop send ./a.txt ./docs --to 192.168.1.23:9090
```

参数：

| 参数             | 当前状态 | 说明                         |
| ---------------- | -------- | ---------------------------- |
| positional paths | 已实现   | 待发送文件或目录，可传多个   |
| `--to`           | 已实现   | 目标地址，格式为 `host:port` |

当前限制：

- 尚未支持 `--resume`；
- 尚未支持 `--chunk-size`；
- 尚未支持 `--config`；
- 尚未显示传输进度；
- 目录会被递归扫描为文件列表；空目录不会被单独创建。

当前内部会话边界：

- 发送端先发送 `HELLO` packet；
- `HELLO` payload 暂时使用纯文本文件数量；
- 随后连续发送每个文件的 `FILE_INFO` / `DATA` / `FILE_END`；
- 最后发送 `FINISH` packet。

## 4. config

当前状态：**未实现真实配置功能**。

生成默认配置：

```bash
beamdrop config init
```

查看配置：

```bash
beamdrop config show
```

## 5. 演示脚本

接收端：

```bash
build-mingw\beamdrop.exe serve --host 127.0.0.1 --port 9090 --save-dir received
```

发送端：

```bash
build-mingw\beamdrop.exe send .\samples --to 127.0.0.1:9090
```

查看帮助：

```bash
build-mingw\beamdrop.exe
```

运行测试：

```bash
ctest --test-dir build-mingw --output-on-failure
```

## 6. 规划参数

以下参数在设计目标中保留，但当前不可用：

| 参数           | 规划用途           |
| -------------- | ------------------ |
| `--resume`     | 启用断点续传       |
| `--chunk-size` | 指定 DATA 分块大小 |
| `--config`     | 指定配置文件路径   |
